### 2.1 关键文件说明

| 路径 | 作用 |
|------|------|
| `lib/main.dart` | 入口：初始化 Storage、Provider、Router，`WhiteNoiseApp` 使用 `MaterialApp.router` |
| `lib/routes/*` | Navigator 2.0：路径枚举、RouterDelegate、RouteInformationParser |
| `lib/providers/*` | 导航状态、功德状态（与本地存储同步） |
| `lib/services/*` | 音频播放（audioplayers）、本地存储（shared_preferences） |
| `lib/screens/main_shell_screen.dart` | 主壳：底部导航 + 四页 IndexedStack |
| `pubspec.yaml` | 项目名、版本、依赖（provider、audioplayers 6.6.0、shared_preferences）、assets |
| `analysis_options.yaml` | Dart 分析器与 Lint 规则 |
| `android/app/build.gradle.kts` | Android 应用构建配置（Kotlin DSL） |
| `ios/Runner/Info.plist` | iOS 应用信息与权限等 |

## 五、应用入口与当前架构

### 5.1 入口

```dart
// lib/main.dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  final storage = await StorageService.getInstance();
  final navigationProvider = NavigationProvider();
  final meritProvider = MeritProvider(storage: storage);
  final routerDelegate = AppRouterDelegate(navigationProvider: navigationProvider);
  final routeInformationParser = AppRouteInformationParser();
  runApp(WhiteNoiseApp(...));
}
```

### 5.2 根 Widget

- **WhiteNoiseApp**：使用 `MultiProvider` 注入 `NavigationProvider`、`MeritProvider`，根为 `MaterialApp.router`。
- **路由**：Navigator 2.0，`routerDelegate` + `routeInformationParser`，由 `NavigationProvider` 驱动当前 Tab。

### 5.3 主壳与四页

- **MainShellScreen**：底部导航栏 + `IndexedStack` 切换四个子页（SleepScreen、DiyScreen、MeditationScreen、WoodenFishScreen）。
- **SleepScreen**：定时睡眠页占位。
- **DiyScreen**：DIY 白噪音页占位。
- **MeditationScreen**：放松冥想页占位。
- **WoodenFishScreen**：功德页，使用 `MeritProvider` 展示与累加功德数，并持久化到 `StorageService`。

## 十一、扩展开发建议

后续若实现「定时睡眠 / DIY 白噪音 / 冥想 / 木鱼」等功能，可考虑：

| 方面 | 建议 |
|------|------|
| **路由** | 使用 `go_router` 或 `Navigator 2.0` 管理多页与底部导航 |
| **状态管理** | 根据复杂度选用 Provider、Riverpod、Bloc 等 |
| **音频** | 使用 `audioplayers`、`just_audio` 等播放环境音与冥想音频 |
| **本地存储** | 使用 `shared_preferences` 或 `hive` 存设置、功德数等 |
| **资源** | 在 `pubspec.yaml` 的 `flutter.assets` 中声明音频、图片路径 |
| **目录结构** | 如 `lib/screens/`、`lib/widgets/`、`lib/services/`、`lib/models/` 等按职责拆分 |
